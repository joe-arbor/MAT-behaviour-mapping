# Arbor Design System

**Objective:** This design system is built so the design team can easily build and prototype. Known UX/UI issues are accepted; the goal is easy composition and fast prototyping from Figma and the component library, not perfect end-user UX first.

## How we work

- **Compose, don’t invent:** Pages and flows are built by composing existing components from the design system; we avoid one-off components when the library can cover the need.
- **Figma-first:** Designs start in Figma; we use assessments and the create-page flow to turn them into real pages and prototypes.
- **Storybook as the source of truth:** Components are documented in Storybook so designers and developers can see variants, props, and usage in one place.
